# TechPlantStudio ページ改善プロジェクト - コンテキスト

## プロジェクト概要

TechPlantStudioページ（受託制作事業ページ）のブラッシュアップ。
ゴール: **本ページを経て案件の相談数（CVR）を増やす。**

---

## チーム構成

以下の4名で議論を進めている。**ユーザーがOKを出すまでチームは解散しない。**

| 役割 | 視点 |
|------|------|
| **CTO** | 技術的実現可能性、パフォーマンス、実装コスト、コンポーネント設計 |
| **CMO** | マーケティング戦略、CTA最適化、コンバージョン導線、コピーライティング |
| **ペルソナ** | 業務委託案件の発注を検討する企業担当者としての率直なフィードバック |
| **デビルズアドボケート** | 反論・懸念の提起、過大表現リスク、逆効果の可能性の指摘 |

## これまでの作業履歴

### 1. ドキュメント整備（初回セッション）

プロジェクト全体の理解のため、3つのドキュメントをエンジニア・マーケター・デザイナーの3チームで並行作成。

| ファイル | 担当 | 内容 |
|---------|------|------|
| `SPEC.md` | エンジニア | 技術仕様書（スタック、アーキテクチャ、改善案） |
| `PRODUCT.md` | マーケター | プロダクト仕様書（VMV、ペルソナ、競合、ロードマップ） |
| `DESIGN.md` | デザイナー | デザインガイドライン（カラー、タイポグラフィ、コンポーネント） |

### 2. techplantstudio.astro の修正

対応領域セクションの2つの修正を実施済み:
- **AI活用コンサルティングの幅統一**: 別グリッド（max-w-[600px]制限）に分離されていた → 全6項目を1つの `grid-cols-1 md:grid-cols-2 lg:grid-cols-3` グリッドに統合
- **イベント企画・運営の追加**: カレンダーアイコン付きカードを6番目に追加

### 3. 改善提案書の作成

4人チームで議論し、改善提案書を作成。

**ファイル**: `docs/techplantstudio-improvement.md`

#### 特定された主要課題
1. 実績・ポートフォリオセクションが存在しない
2. CTAがページ最下部に1箇所のみ（到達率20-30%）
3. メンバー紹介のテキストが記入途中（これはユーザーが認識済み、問題なし）
4. キャッチコピーが差別化不足（「クリエイティブ×テクノロジー」は汎用的）
5. 料金感・FAQがなく比較検討の土俵に乗れない

#### 合意した改善後のページ構成
```
1. Hero（コピー刷新 + IPA未踏バッジ + CTA）
2. 信頼の証（新設: IPA未踏、NoLogic、コミュニティ規模）
3. 対応領域（得意領域の強調 + 技術ロゴ追加）
4. 中間CTA（新設）
5. 実績・ポートフォリオ（新設）
6. チーム紹介（代表を大きく、他メンバーはコンパクトに）
7. 制作フロー（モバイル改善: 縦線接続、期間目安追加）
8. FAQ / 参考価格（新設）
9. 最下部CTA（コピー改善）
10. Footer
```

### 4. 要件定義書の作成（仕様書駆動開発）

改善提案を5つの要件に分割し、各チームで並行して要件定義書を作成。

| # | ファイル | 要件 | 主な決定事項 |
|---|---------|------|------------|
| 01 | `docs/spec/01-hero-cta.md` | Hero改善 & CTA最適化 | コピー「企画・デザイン・開発、ワンストップで。」、IPA未踏バッジ(amber系で差別化)、CTA3箇所(Hero/中間/最下部)、各CTAに強弱 |
| 02 | `docs/spec/02-trust-section.md` | 信頼の証セクション | タイトル「実績と信頼」(TRACK RECORD)、IPA未踏・NoLogic(β版明記)・コミュニティの3カード、カウントアップアニメーション、「So What?」一文追加 |
| 03 | `docs/spec/03-portfolio.md` | 実績・ポートフォリオ | NoLogic(featured全幅)+コーポレートサイトの2件開始、クライアントワーク枠は実績追加で自動表示、テスティモニアルはコミュニティの声で代替 |
| 04 | `docs/spec/04-faq-pricing.md` | 対応領域改善 & FAQ・価格 | 得意領域(ゲーム/Webアプリ)にバッジ、技術ロゴ15種カテゴリ別、FAQ7問(native HTML details)、参考価格帯6領域 |
| 05 | `docs/spec/05-page-structure-ux.md` | ページ構成・UX改善 | 6→10セクション再構成、Overview→信頼の証に統合、text-gray-400→gray-300、スクロールアニメーション(IO+CSS)、制作フローモバイル縦線、代表プロフィール拡大(160x160)、GA4トラッキング |

## チーム横断の共通方針

- **正直さ・透明性**: β版やステータスは隠さず明記
- **外部ライブラリ追加なし**: IntersectionObserver + CSS、native HTML `<details>` で実装
- **モバイルファースト**: コントラスト改善、フォントサイズ見直し
- **押し売り感の排除**: CTAに強弱、B2B向けの節度ある設計
- **メンバー紹介テキストは記入途中** → 構造変更のみ、テキスト内容は変更しない
