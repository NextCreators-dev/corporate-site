# TechPlantStudio ページ改善プロジェクト - コンテキスト

## プロジェクト概要

TechPlantStudioページ（受託制作事業ページ）のブラッシュアップ。
ゴール: **本ページを経て案件の相談数（CVR）を増やす。**

---

## チーム構成

以下の4名で議論を進めている。**ユーザーがOKを出すまでチームは解散しない。**

| 役割 | 視点 |
|------|------|
| **CTO** | 技術的実現可能性、パフォーマンス、実装コスト、コンポーネント設計 |
| **CMO** | マーケティング戦略、CTA最適化、コンバージョン導線、コピーライティング |
| **ペルソナ** | 業務委託案件の発注を検討する企業担当者としての率直なフィードバック |
| **デビルズアドボケート** | 反論・懸念の提起、過大表現リスク、逆効果の可能性の指摘 |

## これまでの作業履歴

### 1. ドキュメント整備（初回セッション）

プロジェクト全体の理解のため、3つのドキュメントをエンジニア・マーケター・デザイナーの3チームで並行作成。

| `DESIGN.md` | デザイナー | デザインガイドライン（カラー、タイポグラフィ、コンポーネント） |

### 2. techplantstudio.astro の修正

対応領域セクションの2つの修正を実施済み:
- **AI活用コンサルティングの幅統一**: 別グリッド（max-w-[600px]制限）に分離されていた → 全6項目を1つの `grid-cols-1 md:grid-cols-2 lg:grid-cols-3` グリッドに統合
- **イベント企画・運営の追加**: カレンダーアイコン付きカードを6番目に追加

### 3. 改善提案書の作成

4人チームで議論し、改善提案書を作成。

**ファイル**: `docs/techplantstudio-improvement.md`

#### 特定された主要課題
1. 実績・ポートフォリオセクションが存在しない
2. CTAがページ最下部に1箇所のみ（到達率20-30%）
3. メンバー紹介のテキストが記入途中（これはユーザーが認識済み、問題なし）
4. キャッチコピーが差別化不足（「クリエイティブ×テクノロジー」は汎用的）
5. 料金感・FAQがなく比較検討の土俵に乗れない

#### 合意した改善後のページ構成
```
1. Hero（コピー刷新 + IPA未踏バッジ + CTA）
2. 信頼の証（新設: IPA未踏、NoLogic、コミュニティ規模）
3. 対応領域（得意領域の強調 + 技術ロゴ追加）
4. 中間CTA（新設）
5. 実績・ポートフォリオ（新設）
6. チーム紹介（代表を大きく、他メンバーはコンパクトに）
7. 制作フロー（モバイル改善: 縦線接続、期間目安追加）
8. FAQ / 参考価格（新設）
9. 最下部CTA（コピー改善）
10. Footer
```

## チーム横断の共通方針

- **正直さ・透明性**: β版やステータスは隠さず明記
- **外部ライブラリ追加なし**: IntersectionObserver + CSS、native HTML `<details>` で実装
- **モバイルファースト**: コントラスト改善、フォントサイズ見直し
- **押し売り感の排除**: CTAに強弱、B2B向けの節度ある設計
- **メンバー紹介テキストは記入途中** → 構造変更のみ、テキスト内容は変更しない

### 5. トップページ新デザイン実装

`feature/New-design` ブランチで、トップページ（`index.astro`）の全面的なリデザインを実施済み。

#### 実装済みセクション構成
```
1. Hero（コピー刷新 + CTA2つ）
2. News（ニュースセクション、src/data/news.ts からデータ取得）
3. Track Record（実績数値: 200+ゲーム、100+クリエイター、IPA未踏採択）
4. Product: NoLogic（3ステップ説明 + ゲームギャラリー）
5. Community: NextCreators（Discord コミュニティ紹介）
6. Services: TechPlant Studio（受託開発5領域）
7. Vision（代表挨拶 + VMVカード）
8. Company（会社概要テーブル）
9. Footer CTA + Footer
```

#### 新規追加ページ・コンポーネント
- `/news` - ニュース一覧ページ（`src/pages/news/index.astro`）
- `/news/[slug]` - ニュース個別ページ（`src/pages/news/[slug].astro`）
- `/privacy` - プライバシーポリシーページ（`src/pages/privacy.astro`）
- `src/data/news.ts` - ニュースデータ
- `src/data/portfolio.ts` - ポートフォリオデータ
- `src/data/testimonials.ts` - テスティモニアルデータ
- `src/components/TrustSection.astro` - 実績セクションコンポーネント
- `src/components/PortfolioCard.astro` - ポートフォリオカードコンポーネント

### 6. SP版（モバイル）レイアウト修正

Playwright MCP を使用した SP版（375x812）の全セクション目視確認を実施。以下2点を修正。

#### 修正1: フッターリンクの縦並び対応
- **対象**: `src/components/Footer.astro` フッター下部リンク（NoLogic, NextCreators, お問い合わせ, プライバシーポリシー）
- **変更前**: `flex items-center gap-6`（常に横並び → SP で詰まって見づらい）
- **変更後**: `flex flex-col md:flex-row items-center gap-4 md:gap-6`（SP で縦並び、PC で横並び）

#### 修正2: フッターCTAボタン幅の統一
- **対象**: `src/components/Footer.astro` CTAセクションの3ボタン
- **問題**: テキスト長の差でボタン幅がバラバラだった
- **変更**: 全ボタンに `justify-center min-w-[220px]` を追加し、統一された幅で表示
