# 要件定義書: 信頼の証セクション

## 概要

TechPlantStudioページ（受託制作事業ページ `/techplantstudio`）のHeroセクション直下に、「信頼の証」セクションを新設する。IPA未踏採択・NoLogic自社開発・100名超コミュニティ運営という3つの実績を、訪問者（業務委託発注検討企業）に対して誠実かつ効果的に提示し、CVR改善に寄与する。

## 背景・目的

- **課題**: 設立2024年11月の若い会社であり、発注企業から見ると「実績不足では？」という懸念が生じやすい
- **目的**: Heroセクションで興味を持った訪問者が下にスクロールした瞬間に「この会社は信頼できる」と感じる根拠を提示し、離脱を防ぎ、CTAセクションまでの到達率を向上させる
- **KPI**: CTAボタン（制作のご相談はこちら）のクリック率向上

## チーム議論サマリー

### CTO（技術実装観点）

- セクションは `TrustSection.astro` として独立コンポーネント化し、techplantstudio.astro から読み込む構成とする
- 数字のカウントアップアニメーションは Intersection Observer + CSS counter で実装。JavaScript の依存を最小限にする（Astroの島アーキテクチャに適合）
- カードは `grid` レイアウトで、デスクトップ3カラム / タブレット2カラム / モバイル1カラムのレスポンシブ対応
- アイコンはインラインSVGで統一（既存ページの実装パターンに合わせる）。外部フォント依存を避ける

### CMO（マーケティング観点）

- 数字は「大きく、一目で目に入る」ように配置。emerald-500 のアクセントカラーで強調
- IPA未踏は知名度が低いため「経済産業省所管」「国家プロジェクト」というキーワードを補足に含める。これにより権威付けを強化
- 「200+」「100+」など丸い数字+プラス表記で成長を暗示する。端数表記（例: 203件）は逆に「少ない」印象を与えるリスクがあるため避ける
- セクションタイトルは「信頼の証」ではなく「私たちの実績」に変更を提案。「信頼の証」は自称感が強く、発注者には押し付けがましい。ただし英語バッジとして `TRACK RECORD` を使うことでトップページとの一貫性を保つ
- **決定**: セクションタイトルは **「実績と信頼」**、英語バッジは **`TRACK RECORD`** とする

### ペルソナ（発注企業担当者の視点）

- 「IPA未踏」だけ書かれても正直わからない。「経済産業省の国家プロジェクトに採択された」という一文があれば安心感がある
- NoLogicについて「自社プロダクトを持っている = 技術力がある」と受け取れる。ただしベータ版であることを隠されると後から知ったときに不信感になる。正直に書いてほしい
- コミュニティ100名超は「多様なクリエイターとの接点がある = 制作リソースが柔軟」という解釈ができるので好印象
- 数字が並んでいるだけでは「で、何ができるの？」となるため、各カードに1行の「だから何？（So What?）」を入れてほしい

### デビルズアドボケート（リスク指摘）

- **IPA未踏の表現リスク**: 「未踏採択 = 受託品質の保証」と誤解されるような文脈は避けるべき。未踏は個人（代表）の技術力・構想力の証明であり、会社としての受託実績とは別物。対策: 「代表がIPA未踏に採択」と主語を明確にする
- **NoLogicベータ版**: 「200+ゲーム投稿」は事実だが、「商用リリース済みプロダクト」と誤解される表現は避ける。対策: 「β版」を明記する
- **設立間もないことの逆効果**: 実績を並べすぎると「盛っているのでは？」と逆に不信感を招く可能性がある。対策: カードのトーンを控えめにし、事実のみを淡々と述べる。感嘆符や誇張表現は使わない
- **コミュニティ人数**: 「100名超」が本当にアクティブかは不明。対策: 「参加者数100名超」と表記し、アクティブ率への言及は避ける
- **数字の更新運用**: 数字が古くなると逆効果。対策: 実装時にコメントで「要定期更新」と記載する

## 要件詳細

### セクション構成

#### 配置位置
- `techplantstudio.astro` の Hero Section（`</section>` 閉じタグ）直下、Overview Section（`<!-- Overview Section -->`）直上
- スクロールして最初に目に入る位置

#### セクションタイトル
- 英語バッジ: `TRACK RECORD`（既存ページのバッジスタイルに準拠）
- 日本語タイトル: **実績と信頼**
- サブタイトル（任意）: なし（カード内の情報で十分伝わるため）

#### レイアウト（グリッド構成）
- デスクトップ（1024px以上）: 3カラムグリッド
- タブレット（768px〜1023px）: 2カラムグリッド（3つ目のカードは幅100%で下段中央配置）
- モバイル（〜767px）: 1カラム（縦積み）

---

### 各要素の仕様

#### IPA未踏採択

- **メインテキスト**: `IPA未踏IT人材発掘・育成事業2021採択`
- **補足テキスト**: `経済産業省所管のIPAが実施する、突出したIT人材を発掘・育成する国家プロジェクト。代表の技術力と構想力が認められ採択。`
- **アイコン/バッジ**: 盾（Shield）アイコン（SVGインライン）、emerald-500カラー
- **注意点**:
  - 主語は「代表が」であり、「会社として」受託品質を保証する文脈では使わない
  - 「国家プロジェクト」というキーワードで権威付けを補強
  - 外部リンク（IPA公式等）は設置しない（離脱防止）

#### NoLogic自社開発

- **メインテキスト**: `NoLogic 自社プロダクト開発`
- **サブテキスト**: `ノーコードゲーム制作プラットフォーム（β版）`
- **補足テキスト**: `企画・設計・開発・運用まで自社で一気通貫で構築。200件以上のゲームが投稿される実動サービス。`
- **リンク先**: なし（セクション内ではリンクを張らない。詳細を知りたい人はトップページのNoLogicセクションへ）
- **スクリーンショット**: なし（カードレイアウトに収まらないため。将来的にホバーでプレビュー表示は検討可能）
- **注意点**:
  - 「β版」を必ず明記する
  - 「200+ゲーム投稿」は数字セクション側で表示

#### コミュニティ運営

- **メインテキスト**: `100名超のクリエイターコミュニティ運営`
- **補足テキスト**: `エンジニア・デザイナー・モデラー・音楽家・映像クリエイターなど多様な人材が参加。制作リソースの柔軟な確保を実現。`
- **アイコン**: Users / People アイコン（SVGインライン）、emerald-500カラー
- **数字の表示方法**: `100+` を大きく表示（emerald-500、text-4xl以上）
- **注意点**:
  - 「アクティブユーザー数」とは言わない。「参加者数」とする
  - 「制作リソースの柔軟な確保」という受託事業との接続を明示

---

### 数字の見せ方

カード上部に大きな数字を配置し、その下にラベルテキストを置く構成。

| 数字 | サイズ | カラー | ラベル | 対応カード |
|------|--------|--------|--------|------------|
| — | — | — | IPA未踏採択 | IPA未踏カード（数字なし、バッジ表示） |
| 200+ | text-4xl (36px) font-black | emerald-500 | ゲーム投稿数 | NoLogicカード |
| 100+ | text-4xl (36px) font-black | emerald-500 | コミュニティ参加者 | コミュニティカード |

- IPA未踏カードは数字ではなくバッジ（「国家プロジェクト採択」）を強調表示とする
- 数字にはカウントアップアニメーション（0から目標値まで）を付与。Intersection Observer でビューポート内に入った時点で発火
- アニメーション時間: 1.5秒、イージング: ease-out

### コンポーネント構成
```

  components/
    TrustSection.astro    # 新規作成（信頼の証セクション）
  pages/
    techplantstudio.astro  # TrustSectionをimportして配置
```

### 配置箇所（techplantstudio.astro）
```astro
<!-- Hero Section -->
<section>...</section>

<!-- ★ ここに挿入 -->
<TrustSection />

<!-- Overview Section -->
<section>...</section>
```

### カウントアップアニメーション
```javascript
// Intersection Observer でビューポート内に入ったらカウントアップ開始
// <script> タグ内にインラインで記述（Astroの推奨パターン）
// React コンポーネント化は不要（インタラクティブ性が低いため）
```

### 定期更新が必要な数値（コメントで明記）
```astro
<!-- 要定期更新: ゲーム投稿数（現在200+、NoLogicダッシュボードで確認） -->
<!-- 要定期更新: コミュニティ参加者数（現在100+、Discordサーバー設定で確認） -->
```

### 既存ページとの整合性
- トップページ（index.astro）の Metrics Section と数字が重複するが、TechPlantStudioページでは「受託の文脈」で見せるため、補足テキストの切り口が異なる
- デザイントークン（カラー、角丸、フォントサイズ）は既存ページのパターンに完全準拠

## 受け入れ基準

### 機能要件
- [ ] Heroセクション直下に「実績と信頼」セクションが表示される
- [ ] `TRACK RECORD` バッジ + 「実績と信頼」タイトルが表示される
- [ ] IPA未踏カード: メインテキスト・補足テキスト・アイコンが仕様通り表示される
- [ ] IPA未踏カード: 「代表が採択」であることが明確に読み取れる
- [ ] NoLogicカード: 「β版」が明記されている
- [ ] NoLogicカード: 「200+」の数字が表示される
- [ ] コミュニティカード: 「100+」の数字が表示される
- [ ] 補足テキストに「受託事業との関連性（So What?）」が含まれている

### デザイン要件
- [ ] ダークテーマ（#0A0F1C / #111827）に準拠している
- [ ] アクセントカラーがemerald-500 (#10B981) で統一されている
- [ ] 既存ページのカードスタイル（角丸、ボーダー、パディング）と一貫性がある
- [ ] ホバー時にカード枠線が変化する

### レスポンシブ要件
- [ ] モバイル（375px幅）で1カラム表示、テキストが途切れない
- [ ] タブレット（768px幅）で2カラム表示
- [ ] デスクトップ（1280px幅）で3カラム表示

### アニメーション要件
- [ ] 数字のカウントアップアニメーションがビューポート内進入時に発火する
- [ ] アニメーションは1回のみ実行される（再スクロールで再発火しない）
- [ ] アニメーション時間が約1.5秒である

### 品質要件
- [ ] Lighthouse パフォーマンススコアに有意な悪影響がない（90以上を維持）
- [ ] `yarn build` でエラーなくビルドが通る
- [ ] 外部JSライブラリの追加依存がない
- [ ] コード内に「要定期更新」コメントが記載されている

### 表現の誠実さ要件
- [ ] IPA未踏を「受託品質の保証」として使っていない
- [ ] NoLogicに「β版」が明記されている
- [ ] 感嘆符や誇張表現（「驚異の」「圧倒的」等）が使われていない
- [ ] 全ての数字が検証可能な事実に基づいている
