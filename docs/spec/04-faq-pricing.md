# 要件定義書: 対応領域改善 & FAQ・参考価格セクション

## 概要

TechPlantStudioページのCVR改善を目的として、以下2つの改修を行う。

1. **対応領域セクションの改善**: 得意領域（ゲーム制作・Webアプリ開発）の視覚的強調、および技術スタック・使用ツールのロゴ一覧を追加
2. **FAQ / 参考価格セクションの新設**: 制作フローセクションの後、最下部CTAの前に配置。よくある質問と参考価格帯を掲載

## 背景・目的

- TechPlantStudioは2024年11月設立の受託制作サービスであり、問い合わせ獲得（CVR向上）が最重要KPI
- 現状ページでは6つの対応領域が均等に並んでおり、得意領域が伝わりにくい
- 料金・納期の情報がないため、問い合わせ前に離脱するユーザーが多いと推測される
- 技術スタックの明示がなく、技術選定に関心のある発注担当者への訴求が弱い

## チーム議論サマリー

### CTO（技術視点）

- **アコーディオンUI**: Astroの `<details>` / `<summary>` ネイティブHTML要素で実装し、JSを不要にする。アニメーションはCSS `transition` + `grid-template-rows` トリックで対応可能。Reactコンポーネント化は不要（パフォーマンス優先）
- **技術ロゴの管理**: `/public/logos/` ディレクトリにSVGファイルを配置。ロゴデータは配列で管理し、map で展開する。将来的にCMSに移行しやすい構造にする
- **得意領域の強調**: ゲーム制作とWebアプリ制作のカードに `border-emerald-500/40` を適用し、左上に「得意領域」バッジを追加。他の4領域は現行の `border-[#1E293B]` を維持
- **レスポンシブ**: 技術ロゴはモバイルで3列、タブレットで4列、PCで6-8列のグリッド。FAQ はモバイルでもタップしやすいよう最低48pxのタッチターゲットを確保

### CMO（マーケティング視点）

- **価格表示戦略**: 「参考価格」として幅を持たせる表記にし、「最終的なお見積もりは無料ヒアリング後に提示」と明記。これにより問い合わせのハードルを下げつつ、価格がひとり歩きするリスクを軽減
- **FAQ設計**: ユーザーの検討フェーズに合わせた質問順序にする。まず「何ができるか」→「いくらかかるか」→「どう進むか」→「契約形態」の流れ
- **得意領域の訴求**: ゲーム制作は「NoLogicという自社プロダクトを持っている」点、Webアプリは「企画〜運用まで一気通貫」という点をバッジやサブテキストで強調
- **技術ロゴの効果**: 技術スタックを見せることで「ちゃんとした技術力がある」という信頼感を醸成。特にReact, Next.js, TypeScript等のモダン技術は発注担当者への安心材料になる

### ペルソナ（業務委託発注企業担当者視点）

- **料金感の妥当性**: Webサイト30万円〜は妥当だが、Webアプリ50万円〜は安く見えすぎて品質不安を感じる可能性がある。ただし「〜」がついていれば「最低ライン」と理解できる
- **知りたい情報**: 一番知りたいのは「自分のプロジェクトがいくらで、どのくらいの期間でできるか」。参考価格があると社内稟議の概算として使えるので助かる
- **対応領域の広さ**: 6つあると「本当に全部できるの？」と不安になるが、得意領域が明示されていれば「ここが強みで、他もカバーできる」と理解できる
- **技術スタック**: 技術に詳しくない担当者でも、有名なロゴ（React, Unity等）が並んでいると安心感がある。ロゴだけでなくカテゴリ分け（フロントエンド / ゲーム / デザイン等）があると分かりやすい

### デビルズアドボケート（批判的検証）

- **価格表示のリスク**: 競合に価格を知られる懸念はあるが、設立間もないスタートアップでは「問い合わせが来ない」ことの方がリスクが大きい。価格を出さない競合との差別化にもなる
- **価格の下限表記の罠**: 「30万円〜」と書いて実際は100万円のケースが多いと、信頼を失う。FAQ内で「規模による変動要因」を丁寧に説明すべき
- **領域の広さへの懸念**: 6領域はスタートアップとしてリソースが分散している印象を与える。得意領域を明確にすることで「選択と集中ができている」メッセージを発信すべき。また、各領域カードの説明文で自社プロダクトやクリエイターとの連携に言及し、「なぜこの領域に対応できるか」の裏付けを示す
- **アコーディオンの注意点**: 閉じた状態だと情報が見えないため、SEO的に不利にならないか確認が必要。HTML `<details>` はGoogleがコンテンツを認識するので問題なし

### 議論の結論

| 論点 | 結論 |
|---|---|
| 価格を出すか | 出す。「参考価格」として幅を持たせ、注意書きを添える |
| 価格帯の設定 | 各領域の下限価格を表示。Webアプリは50万円〜を維持（最低ラインとして妥当） |
| 得意領域の強調方法 | ゲーム制作・Webアプリ制作の2つを視覚的に差別化（ボーダー色+バッジ） |
| 6領域の見せ方 | 得意2領域を上段に配置し強調、残り4領域は控えめに表示 |
| アコーディオンの実装 | ネイティブ `<details>` + CSSアニメーションで実装 |
| 技術ロゴの表示 | カテゴリ別にグルーピングし、SVGロゴを表示 |

## 要件詳細

### 1. 対応領域セクション改善

#### 1.1 得意領域の強調

**強調対象（2領域）:**

| 領域 | 強調理由 |
|---|---|
| ゲーム制作 | 自社プロダクト「NoLogic」を持つ。ゲームエンジニア・CGモデラーが在籍 |
| Webアプリ制作 | コア技術領域。企画〜運用の一気通貫対応が強み |

**視覚的な差別化方法:**

- **得意領域カード（ゲーム制作・Webアプリ制作）:**
  - ボーダー: `border-emerald-500/40`（現行のゲーム制作カードと同等の強調）
  - 左上バッジ: 「得意領域」ラベル（`bg-emerald-500/15 text-emerald-500 text-[11px] font-bold px-3 py-1 rounded-full`）
  - カード内にアイコンの背景として `bg-emerald-500/10` の円を追加
- **その他の領域カード（4領域）:**
  - 現行デザインを維持: `border-[#1E293B]`
  - 変更なし

**レイアウト変更:**

- 現行: 3列均等グリッド
- 改修後: 上段に得意領域2枚を大きめに配置（`md:grid-cols-2`）、下段に残り4枚を小さめに配置（`md:grid-cols-2 lg:grid-cols-4`）
- モバイル: 全て1列で得意領域カードが先に表示

#### 1.2 技術スタック一覧

対応領域セクションの下部に「技術スタック・使用ツール」サブセクションとして配置。

**掲載する技術・ツール（カテゴリ別）:**

| カテゴリ | ツール・技術 |
|---|---|
| フロントエンド | React, Next.js, Astro, TypeScript, Tailwind CSS |
| バックエンド | Node.js, Python |
| ゲーム・3D | Unity, Blender |
| デザイン | Figma, Adobe Creative Suite |
| AI | OpenAI API, Claude API |
| インフラ | Vercel, AWS, Firebase |

**表示形式:**

- 各技術のSVGロゴ（モノクロ / グレースケール基調、ホバー時にカラーに変化）
- ロゴの下にテキストラベル（技術名）
- カテゴリ見出しはテキストのみ（`text-gray-500 text-xs font-semibold tracking-wider uppercase`）
- グリッドレイアウト: モバイル3列 / タブレット4列 / PC6列
- ロゴサイズ: 40x40px（統一）
- ロゴ配置: `/public/logos/` ディレクトリにSVGファイルとして格納

**ロゴデータ管理:**

```typescript
// 型定義
interface TechItem {
  name: string;       // 表示名
  logo: string;       // SVGファイルパス（/logos/react.svg 等）
  category: string;   // カテゴリ名
}
```

ページ内で配列として定義し、`map` で展開する。

**UIコンポーネント（アコーディオン仕様）:**

- HTML: `<details>` / `<summary>` 要素を使用（JavaScript不要）
- 初期状態: 全て閉じた状態
- 開閉アニメーション: CSS `grid-template-rows: 0fr → 1fr` のトランジション（300ms ease）
- アイコン: `<summary>` 右端にシェブロンアイコン（`▼`）、開閉時に180度回転
- スタイル:
  - 各項目間にボーダー: `border-b border-[#1E293B]`
  - `<summary>` のスタイル: `py-5 px-6 text-white font-semibold text-base cursor-pointer flex justify-between items-center`
  - 回答部分のスタイル: `px-6 pb-5 text-gray-400 text-sm leading-relaxed`
  - ホバー時: `<summary>` に `hover:text-emerald-400` を適用
- アクセシビリティ: ネイティブ `<details>` 要素によりキーボード操作・スクリーンリーダー対応が標準で実現

### 技術ロゴのSVG取得

- 各技術の公式SVGロゴを使用（ライセンス確認の上）
- Simple Icons（https://simpleicons.org/）から取得可能なものはそちらを利用
- サイズは40x40pxに統一、`fill` を `currentColor` に統一してTailwindでカラー制御

- SVGロゴはインライン化せず、`<img>` タグの `loading="lazy"` で遅延読み込み
- アコーディオンはJS不使用のため、バンドルサイズへの影響なし
- 新規npm パッケージの追加なし
