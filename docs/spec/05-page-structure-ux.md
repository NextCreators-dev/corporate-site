# 要件定義書: ページ全体構成・UX改善

## 概要

TechPlantStudioページ（`/techplantstudio`）のCVR改善を目的として、セクション順序の再構成、UX改善、GA4イベントトラッキングの3つの要件を定義する。

- 技術スタック: Astro + Tailwind CSS + React
- テーマ: ダーク（`#0A0F1C` / `#111827`）、アクセントカラー: `emerald-500`（`#10B981`）

## 背景・目的

現在のTechPlantStudioページは、Hero → Overview → 対応領域 → メンバー → 制作フロー → CTA の順で構成されている。しかし以下の課題がある:

1. **信頼性の訴求が弱い**: IPA未踏採択など客観的な実績の提示がない
2. **CTAが最下部のみ**: ページ途中で離脱するユーザーを取りこぼしている
3. **実績・ポートフォリオがない**: 発注検討者が判断材料を得られない
4. **FAQ・価格感がない**: 問い合わせのハードルが高い
5. **テキスト可読性**: `text-gray-400`はダーク背景に対してコントラスト比が不十分（WCAG AA未達の箇所あり）
6. **モバイルでの制作フロー表示**: ステップ間の関連性が視覚的に伝わらない
7. **計測基盤がない**: GA4イベントが未実装で改善PDCAが回せない

## チーム議論サマリー

### CTO（技術視点）

- **スクロールアニメーション**: IntersectionObserverをAstroのクライアントサイドスクリプト（`<script>`タグ）で実装する。Reactコンポーネント化は不要。`threshold: 0.1`で発火し、CSSクラスのトグルで制御する。Framer Motionなどの外部ライブラリは不使用とし、バンドルサイズへの影響をゼロにする。
- **GA4実装**: `gtag()`をラップしたユーティリティ関数を作成。CTAクリックは`onclick`、セクションimpressionはIntersectionObserver（アニメーションと同一のオブザーバーで兼用可能）、スクロール深度はGA4の自動計測（Enhanced Measurement）を有効化することで対応する。
- **パフォーマンス**: アニメーションは`transform`と`opacity`のみ使用し、レイアウトシフト（CLS）を発生させない。`will-change`の乱用は避ける。

### CMO（マーケティング視点）

- **ページ導線の設計意図**: ユーザーの心理ステップに沿ったセクション配置を提案。Hero（興味喚起）→ 信頼の証（安心感）→ 対応領域（「自分事化」）→ 中間CTA（即行動層の獲得）→ 実績（検討材料）→ チーム（人柄・安心感）→ フロー（手順の透明性）→ FAQ/価格（不安解消）→ 最下部CTA（最終プッシュ）。
- **Overviewセクションの統合**: 現在のOverview（「TechPlantStudioとは」）の文章は信頼の証セクションの導入文として吸収し、3つの強みカード（ワンストップ対応・クリエイティブ×テクノロジー・スピード感ある納品）は信頼の証セクション内に配置する。独立セクションとしては廃止。
- **中間CTAの重要性**: スクロール深度50%付近（対応領域の直後）にCTAを配置。テキストは「対応領域を見て興味を持った層」に向けたものにする。

### ペルソナ（業務委託発注企業担当者の視点）

- **情報の優先順位**: まず「何ができるか」より「信頼できるか」を知りたい。IPA未踏採択、法人格があること、メンバーのバックグラウンド（元ゲーム会社等）が早い段階で見えると安心する。
- **価格感**: 具体的な金額でなくても「参考価格帯」や「Webサイト制作: 30万円〜」のようなレンジがあると、問い合わせ前のスクリーニングができてありがたい。
- **FAQ**: 「最低発注金額は？」「納期はどれくらい？」「修正回数は？」「契約形態は？」といった質問に答えてほしい。
- **テキストの読みやすさ**: 現状のグレーテキストは少し薄くて読みづらい。特にスマホで見ると目が疲れる。

### デビルズアドボケート（批判的視点）

- **アニメーション過多のリスク**: 全セクションにフェードインを入れると「演出過多」に感じる。初見時のみ発火、速度は控えめ（300-400ms）、同時に複数要素がアニメーションする場合はスタガー（ずらし）を入れる。
- **セクション数の増加**: 10セクションは多すぎないか？→ 各セクションの役割が明確で、1セクションあたりの情報量が適切なら問題ない。ただし、信頼の証セクションはOverviewと統合するため実質的には現在の6→9セクションへの増加（+3）。
- **FAQ/価格セクションのリスク**: 価格を出すことで「安い」「高い」の判断だけで離脱されるリスクがある。→「参考価格」と明記し、「プロジェクトにより変動します。まずはご相談ください」の文言を添える。
- **GA4のパフォーマンス影響**: IntersectionObserverの乱用は避ける。1つのオブザーバーで全セクションを監視する設計にする。

### 合意事項

1. セクション順序は提案通り10セクション構成で進める
2. Overviewは信頼の証に統合し、独立セクションとしては廃止
3. アニメーションはCSS + IntersectionObserverで軽量に実装
4. GA4はgtagラッパー + IntersectionObserver（アニメーションと兼用）で実装
5. テキストカラーは`text-gray-300`をベースとし、補足的な情報のみ`text-gray-400`を許容
6. メンバー紹介のテキスト内容は現状維持（構造変更のみ）

### 2. UX改善

#### 2.1 テキストカラー改善

**目的**: ダーク背景に対するテキストの可読性向上（WCAG AA準拠を目指す）

| 変更箇所 | 変更前 | 変更後 | 備考 |
|----------|--------|--------|------|
| 本文テキスト（説明文、段落） | `text-gray-400`（`#9CA3AF`） | `text-gray-300`（`#D1D5DB`） | `#0A0F1C`背景に対するコントラスト比: 4.5:1以上を確保 |
| 補足テキスト（キャプション等） | `text-gray-400`（`#9CA3AF`） | `text-gray-400`（据え置き） | 補足的情報のみ許容。ただしフォントサイズ14px以上を確保 |
| カード内説明文（`text-[13px]`） | `text-gray-400` + `text-[13px]` | `text-gray-300` + `text-sm`（14px） | 小さいフォントサイズではコントラスト比の要件が厳しくなるため、サイズも引き上げ |

**変更対象ファイル**: `src/pages/techplantstudio.astro`

**変更方針**:
- `text-gray-400`を`text-gray-300`に一括置換するのではなく、セクションごとに意味的な役割を確認して適用
- Heroセクションのサブコピーは`text-gray-300`に変更
- 各カード内の説明文は`text-gray-300`に変更し、`text-[13px]`は`text-sm`に変更

#### 2.2 スクロールアニメーション

**目的**: セクション遷移時の視覚的フィードバックでユーザーエンゲージメントを向上

##### 対象要素とアニメーション種類

| 対象 | アニメーション | 方向 | 備考 |
|------|--------------|------|------|
| セクションタイトル（h2） | フェードイン + 上からスライド | ↑ 20px | `translateY(20px)` → `translateY(0)` |
| セクション説明文 | フェードイン | - | タイトルから100msディレイ |
| カード群（対応領域、強みカード等） | フェードイン + 上からスライド（スタガー） | ↑ 30px | 各カード100msずつずらし |
| CTAボタン | フェードイン | - | セクション内の最後に発火 |
| 制作フローのステップ | フェードイン + 左からスライド（スタガー） | ← 20px | 各ステップ150msずつずらし |
| チーム紹介のメンバーカード | フェードイン（スタガー） | - | 各カード80msずつずらし |

##### 実装方法

```
実装場所: src/pages/techplantstudio.astro 内の <script> タグ
ライブラリ: 不使用（Vanilla JS + CSS）
```

**CSSクラス定義**（`<style>`タグまたはTailwindの`@layer`）:

```css
/* 初期状態（非表示） */
.animate-on-scroll {
  opacity: 0;
  transform: translateY(20px);
  transition: opacity 400ms ease-out, transform 400ms ease-out;
}

.animate-on-scroll.slide-left {
  transform: translateX(-20px);
}

/* 表示状態 */
.animate-on-scroll.is-visible {
  opacity: 1;
  transform: translateY(0) translateX(0);
}

/* スタガー用ディレイ */
.animate-on-scroll[data-delay="1"] { transition-delay: 100ms; }
.animate-on-scroll[data-delay="2"] { transition-delay: 200ms; }
.animate-on-scroll[data-delay="3"] { transition-delay: 300ms; }
.animate-on-scroll[data-delay="4"] { transition-delay: 400ms; }
.animate-on-scroll[data-delay="5"] { transition-delay: 500ms; }
```

**JavaScript実装**:

```javascript
// IntersectionObserverの設定
const observer = new IntersectionObserver(
  (entries) => {
    entries.forEach((entry) => {
      if (entry.isIntersecting) {
        entry.target.classList.add("is-visible");
        // GA4セクションimpression計測（兼用）
        const sectionName = entry.target.closest("section")?.dataset.section;
        if (sectionName && entry.target.dataset.trackImpression !== undefined) {
          gtag("event", "section_impression", { section_name: sectionName });
        }
        observer.unobserve(entry.target); // 一度だけ発火
      }
    });
  },
  { threshold: 0.1 }
);

document.querySelectorAll(".animate-on-scroll").forEach((el) => {
  observer.observe(el);
});
```

**注意事項**:
- `prefers-reduced-motion: reduce`メディアクエリに対応し、アニメーションを無効化する
- アニメーションは`opacity`と`transform`のみ使用（CLSを発生させない）
- `will-change`プロパティは使用しない（ブラウザの自動最適化に任せる）

#### 2.3 制作フロー モバイル改善

**現状の問題**: モバイル表示時、ステップ間の矢印（`hidden md:flex`）が非表示になり、ステップ間の関連性が視覚的に伝わらない。

**改善内容**:

##### 縦線接続の仕様

- モバイル表示時（`md`ブレークポイント未満）、各ステップの間に縦線を表示
- 縦線はステップの番号円（`w-12 h-12 bg-emerald-500 rounded-full`）の中心から下のステップの番号円の中心まで接続
- 実装方法: 各ステップを`relative`で配置し、`::after`擬似要素で縦線を描画

```
縦線の仕様:
- 幅: 2px
- 色: emerald-500/30（#10B981の30%不透明度）
- 位置: ステップ番号円の中心下端から次のステップ番号円の中心上端まで
- 最後のステップには縦線なし
- md以上のブレークポイントでは非表示（横矢印に切り替え）
```

##### 期間目安の表示

各ステップに期間目安テキストを追加:

| ステップ | 期間目安テキスト |
|---------|---------------|
| 1. ヒアリング | 約1週間 |
| 2. 企画・設計 | 1〜2週間 |
| 3. 制作・開発 | 2〜8週間（規模により変動） |
| 4. 納品・運用 | 納品後もサポート |

表示仕様:
- ステップタイトルの下、説明文の上に配置
- スタイル: `text-emerald-500/70 text-xs font-medium`
- フォーマット: 「目安: {期間}」

### 3. GA4イベントトラッキング

#### イベント一覧

| # | イベント名 | トリガー | パラメータ | 備考 |
|---|-----------|---------|-----------|------|
| 1 | `cta_click` | CTAボタンクリック時 | `cta_location`: `hero` / `middle` / `bottom` | 3箇所のCTAを区別 |
| 2 | `section_impression` | セクションがビューポートに10%以上表示 | `section_name`: セクション識別子 | IntersectionObserverで検知。各セクション1回のみ |
| 3 | `scroll_depth` | ページスクロール | `percent_scrolled`: 25 / 50 / 75 / 90 | GA4のEnhanced Measurementで自動計測。追加実装不要 |
| 4 | `faq_expand` | FAQアコーディオン開閉時 | `faq_question`: 質問テキスト | FAQ新設セクション用 |

#### セクション識別子（`data-section`属性値）

| セクション | `data-section`値 |
|-----------|-----------------|
| Hero | `hero` |
| 信頼の証 | `trust` |
| 対応領域 | `services` |
| 中間CTA | `cta_middle` |
| 実績・ポートフォリオ | `portfolio` |
| チーム紹介 | `team` |
| 制作フロー | `flow` |
| FAQ / 参考価格 | `faq_pricing` |
| 最下部CTA | `cta_bottom` |

#### 実装方法

```
実装場所: src/pages/techplantstudio.astro 内の <script> タグ
前提: GA4タグ（gtag.js）がLayout.astroまたはHead.astroで読み込み済みであること
```

**gtagラッパー関数**:

```javascript
// GA4イベント送信ユーティリティ
function trackEvent(eventName, params = {}) {
  if (typeof gtag === "function") {
    gtag("event", eventName, params);
  }
}
```

**CTAクリックトラッキング**:

```javascript
// 各CTAボタンにdata-cta-location属性を付与
document.querySelectorAll("[data-cta-location]").forEach((el) => {
  el.addEventListener("click", () => {
    trackEvent("cta_click", {
      cta_location: el.dataset.ctaLocation,
    });
  });
});
```

**セクションimpressionトラッキング**:

```javascript
// スクロールアニメーション用のIntersectionObserverと兼用
// 各sectionタグにdata-section属性を付与
const sectionObserver = new IntersectionObserver(
  (entries) => {
    entries.forEach((entry) => {
      if (entry.isIntersecting) {
        const sectionName = entry.target.dataset.section;
        if (sectionName) {
          trackEvent("section_impression", { section_name: sectionName });
        }
        sectionObserver.unobserve(entry.target);
      }
    });
  },
  { threshold: 0.1 }
);

document.querySelectorAll("section[data-section]").forEach((el) => {
  sectionObserver.observe(el);
});
```
