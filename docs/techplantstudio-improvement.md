# TechPlantStudio ページ改善提案書

## 議論参加者
- **CTO（最高技術責任者）**: 技術的な実現可能性、パフォーマンス、実装コストの観点から発言
- **CMO（最高マーケティング責任者）**: マーケティング戦略、CTA最適化、コンバージョン導線の観点から発言
- **ペルソナ（企業の業務委託発注担当者）**: 実際に発注を検討する立場からの率直なフィードバック
- **デビルズアドボケート**: あえて反論・懸念を提起し、議論の質を高める役

## エグゼクティブサマリー

現在のTechPlantStudioページは、サービス概要と対応領域の情報は網羅されているものの、**CVR（案件相談数）を高めるための設計が不十分**である。主要な改善方向は以下の通り：（1）実績・ポートフォリオセクションの新設による信頼性強化、（2）CTAの複数箇所配置とマイクロコピーの最適化、（3）ファーストビューでの価値提案の明確化、（4）料金感・FAQ等の意思決定支援コンテンツの追加、（5）メンバー紹介の品質向上（説明文の修正・写真の追加）。設立1年の若い会社であるからこそ、IPA未踏採択の権威性と自社プロダクト（NoLogic）の存在を最大限に活用し、「若さ＝スピードと柔軟性」というポジティブな文脈に転換する必要がある。

## 議論詳細

### 論点1: ページ構成・情報設計

**CTO**:
現在の構成（Hero → Overview → 対応領域 → メンバー → 制作フロー → CTA）は、基本的な情報は揃っている。ただし技術者の視点で言うと、**実績・ポートフォリオのセクションが完全に欠落している**のが致命的。発注者が最も知りたいのは「何ができるか」ではなく「何をやったか」。また、ページのパフォーマンス面では、Hero画像の最適化が必要。現状はhero-bg.pngを1920x1080で読み込んでいるが、Astroの画像最適化をもっと活かせる。

**CMO**:
ファーストビューが弱い。「TechPlantStudio」というサービス名と「クリエイティブ×テクノロジーで、御社のプロジェクトを形にする。」というコピーだけでは、**訪問者は3秒以内に「自分に関係あるか」を判断できない**。ファーストビューに以下を入れるべき：
- 具体的な成果（数字）：「プロジェクト実績○件」「納品率100%」など
- 信頼の証：IPA未踏採択バッジ
- 即座のCTA：ボタンをHeroにも配置

情報設計の順序としては、**信頼性→サービス→実績→フロー→CTA** の流れが理想。現在の「Overview（自社紹介）」が先に来るのは自己中心的で、顧客視点になっていない。

**ペルソナ**:
正直に言うと、このページを見た第一印象は「きれいだけど、実績がわからない」。私が発注を検討する際に最も重要視するのは：
1. **過去の実績**（どんな案件をやったか、クオリティはどうか）
2. **料金感**（概算でいいので相場感がわからないと問い合わせすらしない）
3. **レスポンスの速さ**（問い合わせ後、いつ返事が来るか）

現状はこの3つとも見えない。「対応領域」は広く書いてあるが、**広すぎて逆に不安**。設立1年でこの6領域すべて本当に対応できるのかと疑問に感じる。

**デビルズアドボケート**:
CMOの「信頼性を先に」という提案に反論。**設立1年で実績が少ない場合、信頼性セクションを先に出すと逆効果になるリスクがある**。「実績3件」と書いたら「たった3件？」と思われる。まだ実績が少ない段階では、先に「何ができるか（ケイパビリティ）」と「どうやるか（プロセス）」を見せ、「この会社なら任せられそう」と感じさせてから実績を見せる方が良いのでは？また、対応領域が6つあるのは「広すぎて不安」とペルソナは言ったが、**逆に絞りすぎると機会損失**になる。この会社の強みはマルチクリエイター集団であることなのだから、「広さ」自体は武器にもなりうる。

**CMO（反論）**:
デビルズアドボケートの指摘は鋭い。実績が少ないなら「実績○件」ではなく、**IPA未踏採択・NoLogic開発・100名超コミュニティ運営**という「自社の実績」を前面に出す。これらは外部案件の実績ではないが、技術力とプロジェクト遂行能力の証明になる。「クライアントワーク実績」と「自社プロダクト実績」を分けて見せればいい。

**CTO（補足）**:
同意。NoLogicのスクリーンショットや実際の動作デモへのリンクを貼れば、それ自体が強力なポートフォリオになる。技術力の証明として最も説得力がある。

**結論**:
- ファーストビューにIPA未踏採択バッジ、具体的な数字、CTAボタンを追加
- 実績・ポートフォリオセクションを新設（自社プロダクト実績 + クライアントワーク実績）
- セクション順序を再構成（後述の「推奨するページ構成」参照）
- 対応領域は6つ維持するが、特に強い領域を視覚的に強調する

### 論点2: CTA・コンバージョン導線

**CMO**:
現在のCTAは**ページ最下部に1箇所のみ**。これは明らかに不十分。離脱率の高いWebページでは、最下部まで到達するユーザーは全体の20-30%程度。改善案：
1. **Hero内にCTAボタン**を配置（ファーストビューCTA）
2. **対応領域セクション直後**にCTAを配置（サービス内容を理解した直後）
3. **最下部のCTA**を維持（現状維持）
4. **フローティングCTA**（画面下部に常時表示するバー）の検討

CTAのコピーも「制作のご相談はこちら」は悪くないが、もう少しハードルを下げたい。「まずは無料で相談する」「30分の無料相談を予約する」など、**具体的で心理的ハードルが低い表現**に変更すべき。

**CTO**:
技術的には、フローティングCTAの実装はTailwindのstickyやfixedで簡単にできる。ただし、**モバイルでのフローティングCTAは画面を圧迫する**ので、スクロール量に応じて表示/非表示を切り替えるIntersection Observer実装が望ましい。工数は半日程度。

Google Formsへの遷移は手軽だが、**コンバージョン計測が弱い**。最低限、CTAクリック時にGoogle Analytics（GA4）のイベントを発火させるべき。将来的にはフォームを自前実装してサンクスページでCV計測するのが理想。

**ペルソナ**:
Google Formsに飛ばされると正直「この会社大丈夫かな」と一瞬思う。自社サイト内に問い合わせフォームが埋め込まれている方が信頼感がある。ただ、**問い合わせのハードルとしては、フォームの項目数が重要**。名前・メールアドレス・相談内容の3項目くらいなら気軽に送れる。10項目もあると離脱する。

また、「30分の無料相談」はいいが、**「無料」を強調しすぎると安っぽく見える**。「初回ヒアリング無料」くらいの表現が適切。

**デビルズアドボケート**:
CTAを増やしすぎると**ページが「売り込み感」満載になる**リスクがある。B2Bの発注担当者は押し売りを嫌う。3箇所が上限。フローティングCTAは特にB2Bではウザがられる可能性が高い。

また、CMOの「Hero内にCTA」という提案に疑問。**ファーストビューで何のサービスかもわからない段階でCTAを押すユーザーはいない**。HeroのCTAは「詳しく見る」的なスクロール誘導ボタンにした方が自然では？

**CMO（反論）**:
フローティングCTAについてはデビルズアドボケートに同意、B2Bでは控える。ただしHeroのCTAは「リターン訪問者」向けに有効。初回訪問者はスクロールするが、2回目以降の訪問者はすでにサービス内容を知っているのでHeroから直接問い合わせたい。**両方のボタンを置く（主CTA + 下スクロール誘導）** が正解。

**結論**:
- CTAを3箇所に配置：Heroセクション、対応領域直後、ページ最下部
- HeroのCTAは「制作のご相談はこちら」+ 「サービス内容を見る」の2ボタン構成
- CTAコピーを「初回ヒアリング無料 - まずはご相談ください」に変更
- フローティングCTAは見送り（B2Bには不向き）
- Google Formsは短期的に維持しつつ、中期的に自前フォーム（サイト内埋め込み）に移行
- GAイベントトラッキングを即時実装

### 論点3: 信頼性・実績の訴求

**CMO**:
設立1年の会社にとって信頼性構築は最重要課題。活用できるアセットを整理すると：
1. **IPA未踏IT人材発掘・育成事業 採択** - 国の機関によるお墨付き。これは最大の武器。
2. **NoLogic** - 自社プロダクトの存在は技術力の証明。
3. **100名超のクリエイターコミュニティ運営** - ネットワークの広さと信頼の厚さの証明。
4. **代表の経歴** - IPA未踏に採択されるレベルの人物が代表であること。

これらを「信頼の証」セクションとして、ロゴバッジ・数字・一言説明でコンパクトに見せるべき。

**CTO**:
ポートフォリオセクションについて。現時点でクライアントワークの実績が少なくても、**NoLogicのスクリーンショット・デモリンク、コーポレートサイト自体（このサイト）、コミュニティのDiscordサーバーのスクリーンショット**などを「実績」として掲載できる。「自社プロダクト」「自社サイト」として正直に見せれば問題ない。

将来的にクライアントワーク実績が増えたら、画像カルーセルやケーススタディページに発展させる。Astroのコレクション機能を使えば、実績ページの動的生成も容易。

**ペルソナ**:
IPA未踏は**知っている人には刺さるが、知らない人にはピンとこない**。説明を一言添えるべき。「経済産業省所管のIPAが実施する、突出したIT人材を発掘・育成する国家プロジェクト」くらいの補足があると、その凄さが伝わる。

実績については、**具体的な数字やビフォーアフター**がほしい。「Webサイトを制作しました」だけではなく「制作期間2週間、PV数○%向上」のような情報があると説得力が増す。ただし、初期段階では自社プロダクトの実績でも構わない。

もう一つ、**クライアントの声（テスティモニアル）**があると非常に強い。まだ案件が少ないなら、NoLogicのユーザーの声やコミュニティメンバーの声でも代替可能。

**デビルズアドボケート**:
IPA未踏を前面に出すのは賛成だが、**「未踏に採択された＝受託開発が得意」とはならない**点に注意。未踏はイノベーティブなプロダクト開発を評価するもの。受託案件の信頼性とは別軸。過度に未踏を「受託の品質保証」的に使うとミスリーディングになる。

また、NoLogicを実績として出すのはいいが、**まだリリース前のプロダクトなら「実績」とは言えない**。現在のステータスを正直に書くべき。「開発中」のものを完成品のように見せると、逆に信頼を損なう。

**CMO（応答）**:
的確な指摘。IPA未踏は「技術力と構想力の証明」として使い、「受託品質の保証」とは切り離す。NoLogicも現在のステータスを正直に表記した上で、開発過程で培った技術力をアピールする形にする。透明性こそが信頼に繋がる。

**結論**:
- 「信頼の証」セクションを新設：IPA未踏採択（補足説明付き）、NoLogic開発実績、コミュニティ規模
- IPA未踏の説明は「国（経済産業省所管IPA）が認めた突出IT人材の育成事業に採択」程度の補足を追加
- ポートフォリオセクションにNoLogicのスクリーンショット・デモリンク、自社サイト等を掲載
- クライアントの声（テスティモニアル）セクションを設置（初期はコミュニティメンバーの声で代替可）
- 実績の透明性を重視し、「開発中」「自社プロダクト」等のステータスを正直に表記

### 論点4: コンテンツ・コピーライティング

**CMO**:
現在のキャッチコピー「クリエイティブ×テクノロジーで、御社のプロジェクトを形にする。」は**悪くないが弱い**。理由：
1. 「クリエイティブ×テクノロジー」は抽象的すぎる。何の会社でも言える。
2. 「御社のプロジェクトを形にする」も漠然としている。
3. **差別化要素がコピーに入っていない**。

改善案としては、この会社のユニーク性を反映したコピーにすべき：
- 「エンジニアもデザイナーも、全員クリエイター。」
- 「100名のクリエイターネットワークが、あなたのプロジェクトを動かす。」
- 「IPA未踏採択チームが、企画から開発まで一気通貫。」

サブコピーで補足すれば、具体性と差別化が両立できる。

**CTO**:
コピーの観点ではないが、現在のページに**技術スタックや使用ツールに関する情報がゼロ**。技術選定を判断材料にする発注者もいる。「React / Next.js / Astro / Unity / Blender / Figma」などのロゴを並べるだけでも「ちゃんとモダンな技術を使っている」という安心感を与えられる。

**ペルソナ**:
キャッチコピーより、**具体的に何をいくらでやってくれるのか**の方が知りたい。コピーで感動して問い合わせるわけではない。「クリエイティブ×テクノロジー」と言われても「で、具体的に何？」と思う。

差別化ポイントとして知りたいのは：
1. **他社と比べて何が違うのか**（安いのか、速いのか、クオリティが高いのか）
2. **どんな規模の案件に対応できるのか**（10万円の案件もOK？500万円の大型案件も？）
3. **得意領域はどこか**（6領域すべて「同じくらい得意」はありえない）

**デビルズアドボケート**:
CMOの「100名のクリエイターネットワーク」というコピーに疑問。**100名いること自体は発注者にとって価値ではない**。大手制作会社なら数千人いる。「100名のクリエイターがいるから何がいいのか」を言語化しないと意味がない。例えば「ゲーム・Web・映像・音楽、あらゆるクリエイティブを社内で完結」の方が具体的。

また、ペルソナの「料金感がほしい」に対して、**料金をページに載せるのはリスクもある**。競合に価格を知られる、案件ごとに大きく変わるので誤解を生む、安すぎると見くびられる、など。「料金例」として幅を持たせた表記（例：Webサイト制作 30万円〜）が落としどころか。

**CMO（補足）**:
料金の件は、デビルズアドボケートの懸念に同意しつつ、**「料金例を出さない＝問い合わせが来ない」のが最大のリスク**。B2Bの発注担当者は複数社を比較検討している。料金感がわからない会社は比較対象にすら入れてもらえない。「参考価格」として幅を持たせて出すのがベスト。

**結論**:
- メインキャッチコピーを刷新：具体的な差別化要素を含める
  - 案：「企画から開発、クリエイティブまで。ワンチームで、まるごと。」
  - サブコピー：「エンジニア・デザイナー・映像クリエイター・音楽家が連携し、御社のプロジェクトを一気通貫で実現します」
- 参考価格帯セクション（またはFAQ内で料金感を提示）を追加
- 技術スタック・使用ツールのロゴ一覧を追加
- 得意領域を明確化（ゲーム制作・Webアプリ開発を主軸として視覚的に強調）

### 論点5: UX・デザイン

**CTO**:
技術的な観点からいくつか：
1. **メンバー紹介の説明文に明らかなバグがある**。山縣帆高の説明が「企画・要件定義から開発、。」で途切れている。坂部凌央としゃおの説明文が「AI活用コンサルティング及びLLM〜」で同一テキスト。これは映像クリエイターとイラストレーターの説明ではない。早急に修正すべき。
2. メンバーの**アバター画像がすべて空のグラデーション円**。写真またはイラストのアバターが必要。
3. ページの総読み込み時間は許容範囲だが、Hero画像が重い場合はWebP/AVIF変換を検討。
4. モバイルでは制作フローの矢印が非表示になるのは良い判断だが、**ステップ間の繋がりが視覚的にわかりにくくなる**。縦線やドットで接続すべき。

**CMO**:
デザイン全体について：
1. ダークテーマは制作会社らしくてカッコいいが、**読みやすさは犠牲になっている**。特にtext-gray-400（#9CA3AF）はダーク背景とのコントラスト比が低い。アクセシビリティ基準（WCAG AA）を満たしているか確認すべき。
2. **ビジュアル要素が極端に少ない**。テキストとアイコンだけ。写真、イラスト、スクリーンショット、動画など、視覚的なコンテンツがゼロに近い。クリエイティブ企業のページとして致命的。
3. ページの長さは適切（長すぎず短すぎず）だが、セクション間のリズムが単調。全セクションが同じような「タイトル＋テキスト＋カード」の繰り返し。

**ペルソナ**:
モバイルで見たときの体験として気になるのは：
1. **サービスカードが縦に6つ並ぶ**のはスクロールが長い。アコーディオンやタブ切り替えにした方が見やすい。
2. メンバー紹介が2列で3行になるが、**プロフィール写真がないのでただの文字の塊**に見える。
3. 全体的にテキストが多く、**スキャンしづらい**。見出しを太くするか、重要な数字を大きく見せるなど、メリハリがほしい。

**デビルズアドボケート**:
「ビジュアルが少ない」というのは同意だが、**中途半端なビジュアルを入れるくらいなら、いっそテキスト中心のミニマルデザインに振り切った方がいい**。低品質のストックフォトやフリー素材を入れると逆効果。クリエイティブ企業のポートフォリオ画像として相応しいクオリティのものだけを使うべき。

また、メンバー紹介について。**6名全員を等しく出す必要があるか？** 代表の山縣氏を大きく見せて、他のメンバーは「チーム紹介」として簡潔にする方が、意思決定者（発注者）にとって「誰に話せばいいか」が明確になる。

**CTO（応答）**:
メンバー写真がないのはプライバシーや個人の意向もあるかもしれないが、最低限**アバターイラスト**は用意すべき。しゃおさんがイラストレーターなので、社内で作れるはず。また、アニメーションについては、IntersectionObserverでスクロールに応じたフェードインを実装すると、ページにリズムが生まれる。工数は1日程度。

**結論**:
- 即時修正：メンバー紹介の説明文のバグを修正（山縣氏の文末、坂部・しゃおの説明文の差し替え）
- メンバーのアバター画像を追加（写真またはイラスト、最低限アバターイラスト）
- 代表の山縣氏のプロフィールを大きく・詳しく表示、他メンバーはコンパクトに
- ビジュアル要素の強化：NoLogicのスクリーンショット、制作物の画像等（品質の高いものだけ）
- テキストカラーのコントラスト比を改善（text-gray-400 → text-gray-300に変更検討）
- モバイル対応：サービスカードのアコーディオン化を検討
